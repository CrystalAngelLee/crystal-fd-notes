# [Path](http://nodejs.cn/api/path.html#path)

> 是nodeJS 中的一个内置模块，require之后直接使用
> 主要作用就是用于处理文件\目录的路径

## 常见API

| API                                                          | 功能                   | 说明                                                         |
| ------------------------------------------------------------ | ---------------------- | ------------------------------------------------------------ |
| [basename()](http://nodejs.cn/api/path.html#pathbasenamepath-ext) | 获取路径中基础名称     | 01 返回的就是接收路径当中的最后一部分</br>02 第二个参数表示扩展名，如果说没有设置则返回完整的文件名称带后缀</br>03 第二个参数做为后缀时，如果没有在当前路径中被匹配到，那么就会忽略</br>04 处理目录路径的时候如果说，结尾处有路径分割符，则也会被忽略掉 |
| [dirname()](http://nodejs.cn/api/path.html#pathdirnamepath)  | 获取路径中目录名称     | 返回路径中最后一个部分的上一层目录所在路径                   |
| [extname()](http://nodejs.cn/api/path.html#pathextnamepath)  | 获取路径中扩展名称     | 01 返回 path路径中相应文件的后缀名</br>02 如果 path 路径当中存在多个点，它匹配的是最后一个点，到结尾的内容 |
| [isAbsolute()](http://nodejs.cn/api/path.html#pathisabsolutepath) | 获取路径是否为绝对路径 |                                                              |
| [join()](http://nodejs.cn/api/path.html#pathjoinpaths)       | 拼接多个路径片段       |                                                              |
| [resolve()](http://nodejs.cn/api/path.html#pathparsepath)    | 返回绝对路径           | resolve([from], to)</br>实际应用直接使用path.resolve('index.html') |
| [parse()](http://nodejs.cn/api/path.html#pathparsepath)      | 解析路径               | 01 接收一个路径，返回一个对象，包含不同的信息</br>02 root dir base ext name |
| [format()](http://nodejs.cn/api/path.html#pathformatpathobject) | 序列化路径             | 与 [`path.parse()`](http://nodejs.cn/api/path.html#pathparsepath) 相反 |
| [normalize()](http://nodejs.cn/api/path.html#pathnormalizepath) | 规范化路径             |                                                              |
